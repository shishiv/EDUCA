    The loadStudents function (lines 188-222) may fail for several reasons. Do NOT assume the `.order()` is the problem.

    **Step 1: Add diagnostic logging**

    **Step 2: Improve error handling to show actual error**
    Replace the catch block to log the full error:
    ```typescript

        details: studentsError.details,
        hint: studentsError.hint,

    }
    ```

    **Step 3: Simplify query for reliability**
    Change the query from:
    ```typescript
    const { data, error: studentsError } = await supabase
      .from('matriculas')
      .select(`
        id,
        aluno:alunos(
          id,
          nome_completo,
          nis
        )
      `)
      .eq('turma_id', turmaId)
      .eq('ativo', true)
      .order('aluno(nome_completo)', { ascending: true })
    ```

    To (remove .order(), sort in JS):
    ```typescript
    const { data, error: studentsError } = await supabase
      .from('matriculas')
      .select(`
        id,
        aluno:alunos(
          id,
          nome_completo,
          nis
        )
      `)
      .eq('turma_id', turmaId)
      .eq('ativo', true)
    ```

    **Step 4: Sort results in JavaScript**
    Update the mapping to include sorting:
    ```typescript
    const studentsList: Student[] = (data || [])
      .map((m: any) => ({
        id: m.aluno?.id || '',
        nome: m.aluno?.nome_completo || 'Aluno',
        matriculaId: m.id,
        frequencia: 85, // TODO: Calculate from actual attendance data
        hasNis: !!m.aluno?.nis,
      }))
      .filter((s: Student) => s.id)
      .sort((a, b) => a.nome.localeCompare(b.nome, 'pt-BR'))
    ```

    **Why this approach:**
    - If `ativo` column doesn't exist on `matriculas`, the `.eq('ativo', true)` will fail
    - If RLS blocks the query, we need clear error messages
    - PostgREST ordering on nested relations can be version-dependent
    - Client-side sorting is reliable and works regardless of PostgREST version
