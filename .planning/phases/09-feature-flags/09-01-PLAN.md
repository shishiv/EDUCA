Create SQL migration file with:

1. **feature_flags table** (flag definitions):
   - id: UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - flag_name: TEXT NOT NULL UNIQUE
   - description: TEXT NOT NULL
   - is_active: BOOLEAN DEFAULT true (soft delete support)
   - created_at: TIMESTAMPTZ DEFAULT now()
   - updated_at: TIMESTAMPTZ DEFAULT now()

2. **escola_feature_flags junction table** (per-escola enablement):
   - id: UUID PRIMARY KEY DEFAULT gen_random_uuid()
   - escola_id: UUID NOT NULL REFERENCES escolas(id) ON DELETE CASCADE
   - flag_id: UUID NOT NULL REFERENCES feature_flags(id) ON DELETE CASCADE
   - enabled: BOOLEAN DEFAULT false
   - updated_at: TIMESTAMPTZ DEFAULT now()
   - updated_by: UUID REFERENCES users(id)
   - UNIQUE(escola_id, flag_id)

3. **Indexes** for common queries:
   - idx_escola_feature_flags_escola ON escola_feature_flags(escola_id)
   - idx_escola_feature_flags_flag ON escola_feature_flags(flag_id)
   - idx_feature_flags_name ON feature_flags(flag_name)

4. **RLS Policies**:
   - feature_flags: Authenticated can read active flags
   - feature_flags: Admin/gestor_sme can manage (insert/update/delete)
   - escola_feature_flags: Users can read own escola OR admin can read all
   - escola_feature_flags: Admin/gestor_sme can manage all

5. **Triggers** for updated_at:
   - update_feature_flags_updated_at BEFORE UPDATE
   - update_escola_feature_flags_updated_at BEFORE UPDATE

6. **Seed data** (initial flags, both disabled by default):
   - ('nutricao', 'Modulo de gestao nutricional escolar')
   - ('estoque_escolar', 'Modulo de controle de estoque escolar')

Follow the schema pattern from RESEARCH.md exactly. Use IF NOT EXISTS for idempotent migration.
