# Phase 15.2: Audit Project Configuration - Research

**Researched:** 2026-01-23
**Domain:** Project Configuration (package.json, tsconfig, eslint, next.config)
**Confidence:** HIGH

## Summary

This research investigates the current state of project configuration files and identifies issues, outdated patterns, and best practices for a Next.js 16 + TypeScript 5.9 + ESLint 9 project.

The project has several configuration issues requiring attention:
1. **ESLint configuration is broken** - circular JSON error when running `pnpm lint`
2. **14 outdated dependencies** - including major version upgrades available (Tailwind 4, Zod 4, Zustand 5)
3. **11 unused dependencies** - identified by knip (Radix components, recharts, zustand, etc.)
4. **tsconfig.json uses outdated settings** - `target: "es5"` is very outdated for 2026
5. **next.config.js has deprecated patterns** - turbopack options still under `experimental`

**Primary recommendation:** Fix ESLint configuration first, then audit and update dependencies, then modernize TypeScript and Next.js configurations.

### tsconfig.json Issues

**Current Configuration Problems:**

| Setting | Current | Recommended | Why |
|---------|---------|-------------|-----|
| `target` | "es5" | "es2023" or "esnext" | ES5 is 10+ years outdated, modern browsers support ES2022+ |
| `module` | "esnext" | "nodenext" or "node20" | Better Node.js module resolution |
| `moduleResolution` | "bundler" | "bundler" | Correct for Next.js |
| `jsx` | "react-jsx" | "react-jsx" | Correct |
| `verbatimModuleSyntax` | missing | true | TypeScript 5.9 recommended |
| `noUncheckedIndexedAccess` | missing | true | TypeScript 5.9 recommended |

**Note:** The current tsconfig works, but uses legacy settings. TypeScript 7 (mid-2026) will require more modern settings.

### eslint.config.mjs Issues

**CRITICAL: ESLint is broken**

Running `pnpm lint` produces:
```
TypeError: Converting circular structure to JSON
Referenced from: @eslint/eslintrc
```

**Root Cause:** Using `FlatCompat` with `eslint-config-next` creates circular references in ESLint 9.39.

**Current Configuration:**
```javascript
import { FlatCompat } from '@eslint/eslintrc'
const eslintConfig = [
  ...compat.extends('next/core-web-vitals', 'next/typescript'),
  // ...
]
```

**Recommended Fix:** Use native ESLint 9 flat config with Next.js plugin.

### Recommended eslint.config.mjs (ESLint 9 Flat Config)

```javascript
import { defineConfig, globalIgnores } from 'eslint/config'
import nextPlugin from '@next/eslint-plugin-next'
import tsPlugin from '@typescript-eslint/eslint-plugin'
import tsParser from '@typescript-eslint/parser'

export default defineConfig([
  globalIgnores(['.next/**', 'out/**', 'node_modules/**']),
  {
    files: ['**/*.{ts,tsx}'],
    plugins: {
      '@typescript-eslint': tsPlugin,
      '@next/next': nextPlugin,
    },
    languageOptions: {
      parser: tsParser,
      parserOptions: {
        project: './tsconfig.json',
      },
    },
    rules: {
      // TypeScript rules
      '@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }],
      '@typescript-eslint/no-explicit-any': 'warn',
      // Next.js rules
      '@next/next/no-img-element': 'error',
      // Security
      'no-eval': 'error',
    },
  },
])
```

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Module aliasing | Custom webpack config | tsconfig paths + bundler resolution | Built-in support |
| Dependency updates | Manual version bumping | `pnpm update --interactive` | Safer, easier |
| Dead code detection | Manual grep | knip | Comprehensive, maintained |
| ESLint migration | Manual config rewrite | `@eslint/migrate-config` | Handles edge cases |

## Common Pitfalls

### Pitfall 1: Updating Tailwind to v4 Without Migration

**What goes wrong:** Breaking changes in CSS-first config system, all tailwind.config.js stops working.

**Why it happens:** Tailwind v4 completely changed configuration from JavaScript to CSS.

**How to avoid:** Run `npx @tailwindcss/upgrade` migration tool first. Requires Node.js 20+.

**Warning signs:** Classes not applying, build errors mentioning tailwind config.

### Pitfall 2: Updating Zod to v4 Without Code Changes

**What goes wrong:** Error customization APIs changed significantly.

**Why it happens:** Zod 4 unified error APIs under `error` param, deprecated `message`, `invalid_type_error`.

**How to avoid:** Use codemod `zod-v3-to-v4`, review error handling code.

**Warning signs:** TypeScript errors in schema definitions, validation errors format changed.

### Pitfall 3: ESLint FlatCompat Circular References

**What goes wrong:** `TypeError: Converting circular structure to JSON` when running lint.

**Why it happens:** FlatCompat creates circular references when extending configs with plugins.

**How to avoid:** Use native ESLint 9 flat config format, avoid FlatCompat when possible.

**Warning signs:** Lint command crashes instead of reporting errors.

### Pitfall 4: Moving @types to devDependencies Breaking Builds

**What goes wrong:** Production builds fail with missing types.

**Why it happens:** Some build processes need types at build time.

**How to avoid:** Test `pnpm build` after moving type packages.

**Warning signs:** Build errors about missing type definitions.

### Pitfall 5: Removing "Unused" Dependencies That Are Dynamically Imported

**What goes wrong:** Runtime errors for features that work via dynamic import.

**Why it happens:** knip can't always detect dynamic imports or conditional requires.

**How to avoid:** Verify each "unused" dependency by searching for dynamic imports, checking if it's a peer dependency.

**Warning signs:** recharts used in charts (may be lazy loaded), zustand in stores, vaul for drawer components.
