Create API routes following existing patterns (see app/api/alunos/route.ts):

**app/api/vivencias/route.ts:**
- GET: Accept query params `aluno_id` (required) and `limit` (optional, default 50)
  - Return `{ data: Vivencia[] }` on success
  - Return `{ error: string }` with 400 if missing aluno_id
- POST: Accept body with CreateVivenciaInput
  - Get professor_id from auth session (supabase.auth.getUser())
  - Return `{ data: Vivencia }` with 201 on success
  - Return `{ error: string }` with 400 for validation errors

**app/api/vivencias/[id]/route.ts:**
- GET: Return single vivencia by ID
  - Return `{ data: Vivencia }` on success
  - Return `{ error: string }` with 404 if not found
- PUT: Update vivencia
  - Validate professor owns the vivencia (professor_id matches auth user)
  - Return `{ data: Vivencia }` on success
- DELETE: Remove vivencia
  - Validate professor owns the vivencia
  - Return 204 on success

Use NextResponse from 'next/server' for responses.
All routes should be async functions with proper try/catch.
