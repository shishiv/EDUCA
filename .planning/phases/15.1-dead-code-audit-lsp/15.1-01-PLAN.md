<task type="auto">
  <name>Task 2: LSP audit of lib/api/ and lib/validation/ exports</name>
  <files>gestao_fronteira/lib/api/*.ts, gestao_fronteira/lib/validation/*.ts</files>
  <action>
For each file in lib/api/ (24 files) and lib/validation/ (14 files):

1. Use `mcp__lsp__documentSymbol` to get all exported symbols
2. For each exported symbol, use `mcp__lsp__findReferences` to check usages
3. Flag symbols where findReferences returns only 1 result (the definition itself)

Before removing, verify each flagged symbol:
- NOT a re-export in an index.ts barrel file
- NOT referenced via dynamic import (check `grep -rn "import.*filename"`)
- NOT a Next.js special export (metadata, generateStaticParams, etc.)

For confirmed dead code:
1. Remove the export from the source file
2. If entire file is unused, delete the file
3. Run `pnpm typecheck` to verify no breakage
4. Commit atomically: `refactor: remove unused {symbolName} from lib/{subdir}`

Focus areas (from knip results):
- lib/api/ - API service methods that may be superseded
- lib/validation/ - Validation schemas that may be unused
  </action>
  <verify>
    - All files in lib/api/ and lib/validation/ audited via LSP
    - Dead code removed with atomic commits
    - `pnpm typecheck` passes after each removal batch
    - `pnpm lint` passes
  </verify>
  <done>lib/api/ and lib/validation/ directories audited and cleaned</done>
</task>
