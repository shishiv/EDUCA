<task type="auto">
  <name>Task 2: LSP audit of lib/api/ and lib/validation/ exports</name>
  <files>gestao_fronteira/lib/api/*.ts, gestao_fronteira/lib/validation/*.ts</files>
  <action>
**Step 1: Extract knip candidates for lib/api/ and lib/validation/**
```bash
grep -E "^lib/(api|validation)/" knip-lib-results.txt > knip-api-validation.txt
cat knip-api-validation.txt  # Review candidates
```

**Step 2: For each knip-flagged file, verify with LSP**
For each file in knip-api-validation.txt (prioritize these first):
1. Use `mcp__lsp__documentSymbol` to get all exported symbols
2. For each exported symbol, use `mcp__lsp__findReferences` to check usages
3. Flag symbols where findReferences returns only 1 result (the definition itself)

**Step 3: Verify before removal**
Before removing each flagged symbol:
- Check if re-exported in lib/api/index.ts or lib/validation/index.ts
- Check for dynamic imports: `grep -rn "import.*filename" gestao_fronteira/`
- Verify NOT a Next.js special export (metadata, generateStaticParams, etc.)

**Step 4: Remove dead code with barrel file updates**
For confirmed dead code:
1. Remove the export from the source file
2. **CRITICAL: If symbol was re-exported in index.ts, remove from barrel file too**
   ```bash
   # Check if symbol exists in barrel
   grep "symbolName" lib/api/index.ts
   # If found, remove that export line
   ```
3. If entire file is unused, delete the file AND remove from barrel
4. Run `pnpm typecheck` to verify no breakage
5. Commit atomically: `refactor: remove unused {symbolName} from lib/{subdir}`

**Step 5: Audit remaining files not flagged by knip**
After knip candidates, audit any remaining lib/api/ and lib/validation/ files via LSP to catch edge cases knip missed.
