# Configuração de MCP Servers - SuperClaude

Este projeto usa MCP (Model Context Protocol) servers para expandir as capacidades do Claude Code.

## 📦 MCP Servers Configurados

### ✅ Ativos e Funcionais

1. **Supabase MCP** (`@supabase/mcp-server-supabase`)
   - 🎯 **Propósito**: Operações de banco de dados
   - 🔑 **Requer**: `SUPABASE_ACCESS_TOKEN`
   - ✅ **Status**: Configurado

2. **Chrome DevTools MCP** (`chrome-devtools-mcp`)
   - 🎯 **Propósito**: Validação UI/UX, testes visuais, performance profiling
   - 🔑 **Requer**: Nenhuma configuração adicional
   - ✅ **Status**: Configurado

3. **shadcn-ui MCP** (`@jpisnice/shadcn-ui-mcp-server`)
   - 🎯 **Propósito**: Geração de componentes shadcn/ui
   - 🔑 **Requer**: `GITHUB_API_KEY` (já configurado)
   - ✅ **Status**: Configurado

4. **Context7 MCP** (`@upstash/context7-mcp`)
   - 🎯 **Propósito**: Documentação oficial de frameworks
   - 🔑 **Requer**: API Key Context7 (já configurado)
   - ✅ **Status**: Configurado

5. **Sequential Thinking MCP** (`@modelcontextprotocol/server-sequential-thinking`)
   - 🎯 **Propósito**: Análise multi-passo e raciocínio estruturado
   - 🔑 **Requer**: Nenhuma configuração adicional
   - ✅ **Status**: Configurado

6. **Playwright MCP** (`@executeautomation/playwright-mcp-server`)
   - 🎯 **Propósito**: Automação de navegador e testes E2E
   - 🔑 **Requer**: Nenhuma configuração adicional
   - ✅ **Status**: Configurado

### ⚠️ Requer Configuração Adicional

7. **Tavily MCP** (`@modelcontextprotocol/server-tavily`)
   - 🎯 **Propósito**: Web search e pesquisa em tempo real
   - 🔑 **Requer**: `TAVILY_API_KEY`
   - ⚠️ **Status**: Configurado mas requer API key
   - 🔗 **Obter API Key**: https://app.tavily.com

### 🔍 SuperClaude MCPs Adicionais (Documentados mas não disponíveis como pacotes npm)

8. **Magic MCP** (21st.dev)
   - 🎯 **Propósito**: Geração de componentes UI modernos
   - 📝 **Nota**: Atualmente integrado via comandos `/ui` e `/21`
   - 🔗 **Info**: Veja `~/.claude/MCP_Magic.md`

9. **Morphllm MCP**
   - 🎯 **Propósito**: Edição de código em massa baseada em padrões
   - 📝 **Nota**: Requer instalação manual ou configuração custom
   - 🔗 **Info**: Veja `~/.claude/MCP_Morphllm.md`

10. **Serena MCP**
    - 🎯 **Propósito**: Operações semânticas de código e memória de projeto
    - 📝 **Nota**: Requer instalação manual ou configuração custom
    - 🔗 **Info**: Veja `~/.claude/MCP_Serena.md`

## 🔧 Como Configurar

### 1. Tavily Search (Opcional mas Recomendado)

```bash
# 1. Obtenha sua API key em: https://app.tavily.com
# 2. Edite o arquivo .mcp.json e substitua:
"TAVILY_API_KEY": "tvly-your-api-key-here"
# Por:
"TAVILY_API_KEY": "tvly-SUA-CHAVE-AQUI"
```

### 2. Verificar Instalação

Após reiniciar o Claude Code, verifique se os MCP servers estão ativos:

```bash
# Os seguintes comandos devem estar disponíveis:
# - mcp__supabase__* (operações de banco)
# - mcp__chrome_devtools__* (testes UI/UX)
# - mcp__shadcn-ui__* (componentes UI)
# - mcp__context7__* (documentação)
# - mcp__tavily__* (web search)
# - mcp__playwright__* (automação browser)
# - Thinking tools (sequential reasoning)
```

## 🔄 Atualizações

Todos os MCP servers estão configurados com `@latest` para sempre usar a versão mais recente via `npx -y`.

## 🐛 Troubleshooting

Se um MCP não estiver funcionando:

1. ✅ Reinicie o Claude Code
2. ✅ Verifique se a API key está configurada (Tavily, Context7, etc.)
3. ✅ Confirme que o pacote npm existe: `npx -y <package-name> --help`
4. ✅ Verifique os logs do Claude Code

## 📝 Notas

- **pnpm** é usado para gerenciamento de pacotes do projeto
- **npx** é usado para executar os MCP servers
- Configuração centralizada em: `/c/Repos/SRE/.mcp.json`
- Backup da config global: `~/.claude/.mcp.json.backup`
